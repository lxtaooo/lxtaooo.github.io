Spring Boot

Spring Boot 是一个快速开发框架，可以迅速搭建出一套基于 Spring 框架体系的应用，是 Spring Cloud 的基础。

Spring Boot 开启了各种自动装配，从而简化代码的开发，不需要编写各种配置文件，只需要引入相关依赖就可以迅速搭建一个应用。

  • 特点

1、不需要 web.xml

2、不需要 springmvc.xml

3、不需要 tomcat，Spring Boot 内嵌了 tomcat

4、不需要配置 JSON 解析，支持 REST 架构

5、个性化配置非常简单

##Maven

介绍

项目构建生命周期

  • 简单

    • 开发、编译、打包、发布

    • 打包：简单java工程—》jar包

      ​ 简单的web工程—》war包

  • 专业流程

    清理—>编译—>测试—>报告—>打包—->发布

项目中jar包管理

  • 管理难度
    • 定位较难
    • jar之间依赖较难
    • jar之间冲突
    • jar包管理
  • maven如何管理jar
    • 本地仓库—>私服仓库—->中央仓库

maven命令

  • clean：删除当前工程中target
  • complie：将当前工程中main文件下所有的java编译到class，输送到target文件夹下
  • test：调用test文件下所有测试类的方法，并生成测试报告
  • package：将工程中test文件下所有的java测试类中方法进行调用进行测试，如果测试没有问题则将main文件下所有的class文件打成jar或者war
  • install：与package命令执行基本一致，将jar、war推送到maven本地仓库

maven坐标

  • gav

    为了方便maven对于jar的定位，每个jar都需要有一个独立的坐标，相当于ip，坐标由三个部分组成。

    • <groupid>表示当前jar包所在公司域名的反顺序
    • <artifactid> 表示项目名称
    • <version> 项目版本号

MyBatis

简介

什么是MyBatis？

​ MyBatis是一流的持久性框架，支持自定义SQL，存储过程和高级映射。MyBatis消除了几乎所有的JDBC代码以及参数的手动设置和结果检索。MyBatis可以使用简单的XML或注释进行配置，并将图元，映射接口和Java POJO（普通的旧Java对象）映射到数据库记录。

数据持久化

设置

这是 MyBatis 中极为重要的调整设置，它们会改变 MyBatis 的运行时行为。 下表描述了设置中各项的意图、默认值等。

cacheEnabled 全局地开启或关闭配置文件中的所有映射器已经配置的任何缓存。 true | false true
lazyLoadingEnabled 延迟加载的全局开关。当开启时，所有关联对象都会延迟加载。 特定关联关系中可通过设置 fetchType 属性来覆盖该项的开关状态。 true | false false
logImpl 指定 MyBatis 所用日志的具体实现，未指定时将自动查找。 SLF4J | LOG4J | LOG4J2 | JDK_LOGGING | COMMONS_LOGGING | STDOUT_LOGGING | NO_LOGGING
